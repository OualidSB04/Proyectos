Sobre el servidor: JSONPlaceholder es una API de pruebas. Acepta POST/PUT/DELETE y responde como si guardara los cambios (devuelve el recurso con su nuevo id), pero no persiste de verdad. Es ideal para practicar el lado cliente de un servicio en red.

Conceptos de la unidad

  • API REST: recursos accesibles por URL con verbos HTTP.
  • Métodos: GET (leer), POST (crear), PUT (actualizar), DELETE (borrar).
  • JSON: formato de intercambio entre cliente y servicio.
  • Códigos de estado: 200 OK, 201 Creado, 404 No encontrado…
